<Badge>Recipe only</Badge>

Badge displays non-interactive status text. It is composed directly from an
OpenTUI Box and Text node and distributed only as editable Registry source.
There is no Badge Primitive or Badge subpath in the `@tuiparts/core`,
`@tuiparts/react`, or `@tuiparts/solid` packages.

Use [Button](/catalog/button) when the user can activate the element.

## Install

<CodeGroup>

```bash React
pnpm dlx shadcn@4.13.0 add @tuiparts/react/badge
```

```bash Solid
pnpm dlx shadcn@4.13.0 add @tuiparts/solid/badge
```

```bash Core
pnpm dlx shadcn@4.13.0 add @tuiparts/core/badge
```

</CodeGroup>

## Use the Recipe

### React and Solid

```tsx
import { Badge } from "./components/ui/badge";

<Badge label="Passing" intent="success" />
<Badge label="Deprecated" intent="danger" size="comfortable" />
```

Native Box properties apply to the root. `labelOptions` applies native Text
properties to the label.

### Core

```ts
import { createBadge } from "./components/ui/badge";

const status = createBadge(ctx, {
  label: "Passing",
  intent: "success",
});

parent.add(status);
```

The factory returns a `BoxRenderable` containing the Text label.

## Customize the Recipe

The copied source owns label assembly, intent palettes, horizontal padding,
size choices, and visual defaults. Edit it to add variants or different child
content. Instance Box properties and `labelOptions` override starter values;
shared colors and density come from the [Theme Recipe](/theming).

The framework props and Core options also extend OpenTUI `BoxOptions`.
Instance root properties and `labelOptions` are applied after starter defaults.

## Why Badge is Recipe-only

Primitives exist to version reusable behavior such as state, focus, keys,
collections, and overlays. Badge has none of those responsibilities. Packaging
it would hide presentation without centralizing any difficult behavior.
