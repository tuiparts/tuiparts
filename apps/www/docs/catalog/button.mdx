title: Button
description: A focusable action activated by keyboard, pointer, or imperative code.
---

Button performs an action without retaining an on/off value. The packaged
Primitive owns activation, pressed state, focus, and disabled behavior. The
Recipe adds a label, intent palette, and horizontal density.

Use [Toggle](/catalog/toggle) when the pressed state must remain visible.

## Use the Recipe

### React and Solid

Both framework Recipes export `Button` with the same convenience props:

```tsx
import { Button } from "./components/ui/button";

<Button
  label="Save changes"
  intent="primary"
  size="comfortable"
  onPress={() => save()}
/>;
```

Remaining props pass to the framework Button Primitive, including native
OpenTUI layout properties and the Renderable `ref`.

### Core

The Core Recipe exports an imperative factory:

```ts
import { createButton } from "./components/ui/button";

const button = createButton(ctx, {
  label: "Save changes",
  intent: "primary",
  size: "comfortable",
  onPress: () => save(),
});

parent.add(button);
```

`createButton()` returns the packaged `ButtonRenderable`, so actions such as
`button.press()` remain available.

## Customize the Recipe

The copied source owns the label child tree, intent palette, horizontal
padding, and which native properties its Core factory forwards. Add intent or
size variants directly in that file. Application-wide colors and density come
from the [Theme Recipe](/theming).

## Behavior

| Key | Behavior |
| --- | --- |
| `Enter` or `Return` | Activates the Button. |
| `Space` | Activates the Button. |

An uncancelled primary-pointer release and `press()` request the same action.
Disabled Buttons reject focus and activation.

For transient pressed state, activation details, imperative actions, and raw
composition, see the [Button Primitive](/primitives/button).

## Recipe API

### React and Solid props

<TypeTable
  type={{
    label: {
      type: "string",
      required: true,
      description: "Text rendered inside the Button.",
    },
    intent: {
      type: '"neutral" | "primary"',
      default: '"primary"',
      description: "Uses either the surface or primary theme token pair.",
    },
    size: {
      type: '"compact" | "comfortable"',
      default: '"compact"',
      description: "Selects the horizontal padding token.",
    },
  }}
/>

The Recipe extends `ButtonPrimitive.Props` except for `children`, which the
Recipe owns to render its label.

### Core options

`ButtonOptions` exposes `label`, `intent`, `size`, `disabled`, and `onPress`.
Native presentation options are not forwarded by this starter Core factory;
edit the installed class when you want to expose more.

## Build with the Primitive

Use the packaged [Button Primitive](/primitives/button) when you want to own
the child tree and visual API without starting from the Recipe.
