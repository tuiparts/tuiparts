title: Checkbox

Checkbox represents an independent choice, commonly one of several values
submitted together. The packaged Primitive owns checked state, activation,
focus, and Indicator visibility. The Recipe adds a one-cell mark, label, and
tone.

Use [Switch](/catalog/switch) for a setting that takes effect immediately and
[Radio Group](/catalog/radio-group) for mutually exclusive options.

## React and Solid

```tsx
import { Checkbox } from "./components/ui/checkbox";

<Checkbox
  label="Enable notifications"
  defaultChecked
  onCheckedChange={(checked) => console.log({ checked })}
/>;
```

Use `checked` with `onCheckedChange` for controlled ownership, or
`defaultChecked` for uncontrolled initial state.

## Core

```ts
import { createCheckbox } from "./components/ui/checkbox";

const checkbox = createCheckbox(ctx, {
  label: "Enable notifications",
  defaultChecked: true,
  onCheckedChange: (checked) => console.log({ checked }),
});

parent.add(checkbox);
```

The factory returns `CheckboxRootRenderable`. The installed source constructs
an Indicator attached to the Root's Store.

## Compose the Primitive

Use the Parts directly when the Recipe layout does not fit:

```tsx
import { Checkbox as CheckboxPrimitive } from "@tuiparts/react/checkbox";

<CheckboxPrimitive.Root defaultChecked flexDirection="row" gap={1}>
  {(state) => (
    <>
      <CheckboxPrimitive.Indicator>
        <text content="✓" />
      </CheckboxPrimitive.Indicator>
      <text content={state.checked ? "Enabled" : "Disabled"} />
    </>
  )}
</CheckboxPrimitive.Root>;
```

An authored Indicator remains attached and reflects checked state. Core
callers construct `CheckboxIndicatorRenderable` with the Root's Store.

## Keyboard interaction

| Key | Behavior |
| --- | --- |
| `Enter` or `Return` | Toggles checked state. |
| `Space` | Toggles checked state. |

An uncancelled primary-pointer release and `press()` request the same change.
Disabled Checkboxes reject focus and activation.

React and Solid extend `CheckboxPrimitive.Root.Props` except for `children`.
Core `CheckboxOptions` exposes the same ownership fields, label, and mark, but
does not currently expose the framework Recipe's `tone` convenience option.
