Dialog presents a short modal task or decision. The packaged Primitive owns
open state, renderer-scoped layers, topmost dismissal, focus containment, and
restoration. The Recipe provides a centered popup, backdrop, border, spacing,
and styled controls.

Prefer inline controls when the task does not need to interrupt the current
context. Give every Dialog a visible title and add a description when the
action or consequences need explanation.

## Use the Recipe

### React and Solid

```tsx
import {
  Dialog,
  DialogClose,
  DialogContent,
  DialogDescription,
  DialogTitle,
  DialogTrigger,
} from "./components/ui/dialog";

<Dialog>
  <DialogTrigger>
    <text content="Open settings" />
  </DialogTrigger>
  <DialogContent>
    <DialogTitle content="Settings" />
    <DialogDescription content="Change this project's options." />
    <DialogClose>
      <text content="Close" />
    </DialogClose>
  </DialogContent>
</Dialog>;
```

`DialogContent` composes the Primitive's Portal, Backdrop, and Popup Parts.

### Core

The Core Recipe returns the Parts needed for imperative assembly:

```ts
import { TextRenderable } from "@opentui/core";
import {
  addDialogClose,
  addDialogDescription,
  addDialogTitle,
  createDialog,
} from "./components/ui/dialog";

const dialog = createDialog(ctx);
dialog.trigger.add(new TextRenderable(ctx, { content: "Open settings" }));
addDialogTitle(ctx, dialog, "Settings");
addDialogDescription(ctx, dialog, "Change this project's options.");
addDialogClose(ctx, dialog, "Close");

parent.add(dialog.root);
parent.add(dialog.portal);
```

`createDialog()` returns `root`, `trigger`, `portal`, `backdrop`, and `popup`.
Core callers add both the Root and Portal to the render tree. The helper
functions append semantic text and Close Parts to the Popup.

## Customize the Recipe

The copied source owns the relationship between `DialogContent`, Portal,
Backdrop, and Popup along with popup width, border, spacing, and control
presentation. Edit that source when you need another layer structure or
convenience API. Shared colors and borders come from the
[Theme Recipe](/theming).

## Behavior

Use `open` with `onOpenChange` for controlled ownership, or `defaultOpen` for
uncontrolled initial state. Change details report whether a transition came
from the Trigger, Escape, outside interaction, Close, an accepted action, or a
programmatic request.

Dismissal requests are cancelable where the Primitive can synchronously honor
`details.preventDefault()`.

When a Dialog opens, focus moves to `initialFocus`, the first focusable Popup
descendant, or the Popup itself. Tab and Shift+Tab wrap within the topmost
Popup. Closing restores focus to its Trigger or the appropriate parent Dialog.

Only the topmost eligible layer handles Escape or Backdrop dismissal. Pointer
interaction inside the Popup does not dismiss it.

| Key | Behavior |
| --- | --- |
| `Escape` | Requests dismissal of the topmost Dialog. |
| `Tab` | Moves to the next focusable Popup descendant. |
| `Shift+Tab` | Moves to the previous focusable Popup descendant. |
| `Enter`, `Return`, or `Space` | Activates a focused Trigger or Close Part. |

For all eight Parts, cancelable transitions, and raw layer composition, see
the [Dialog Primitive](/primitives/dialog).

## Recipe API

### Dialog

`Dialog` retains the Primitive Root props: `open`, `defaultOpen`,
`onOpenChange`, native Root properties, children, and the Renderable ref.

### DialogContent

React and Solid `DialogContent` accepts Primitive Popup props and adds
`backdropColor?: ColorInput`. `initialFocus?: Renderable` selects a preferred
focus target. The Recipe owns its Portal and Backdrop tree, so those Part props
are changed by editing the installed source or composing the Primitive.

### DialogClose

`reason` defaults to `"close"`. Set `reason="action"` when activation reports
an accepted Dialog action through `onOpenChange`.

Core `DialogOptions` extends `DialogRootOptions` and adds `width`; the Core
helpers expose the assembled Parts instead of framework Recipe exports.

## Build with the Primitive

Compose the packaged [Dialog Primitive](/primitives/dialog) when you need a
different Portal, Backdrop, or Popup structure than `DialogContent` provides.
