The Catalog contains editable Recipes for Core, React, and Solid. Interactive
Recipes build on packaged Primitives; presentation-only Recipes use OpenTUI
directly.

Install a runtime-specific item with the shadcn CLI:

```bash
pnpm dlx shadcn@4.13.0 add @tuiparts/react/button
```

Replace `react` with `solid` or `core` for the other runtimes. Installation
copies source into `components/ui` and adds the shared consumer-owned theme.

## Choose a Recipe

| Recipe | Use it for | Packaged behavior |
| --- | --- | --- |
| [Button](/catalog/button) | An action without retained on/off state | Activation, pressed state, focus, disabled behavior |
| [Input](/catalog/input) | Single-line text entry | OpenTUI-native editing and events |
| [Checkbox](/catalog/checkbox) | An independent choice commonly submitted with others | Checked state and activation |
| [Switch](/catalog/switch) | A setting that takes effect immediately | Checked state and activation |
| [Toggle](/catalog/toggle) | An action whose pressed state remains visible | Pressed state and activation |
| [Toggle Group](/catalog/toggle-group) | Related toggles sharing one keyboard stop | Selection and roving focus |
| [Radio Group](/catalog/radio-group) | One mutually exclusive selection from a set | Selection and roving focus |
| [Dialog](/catalog/dialog) | A short modal task or decision | Layers, dismissal, and focus containment |
| [Badge](/catalog/badge) | Non-interactive status text | None; Recipe only |

## Recipes are starting points

Each installed file owns its convenience API and composition. React and Solid
Recipes expose JSX functions; Core Recipes expose imperative factory
functions. Catalog pages document that copied source: how to use it, what to
customize, and which convenience API it provides.

Application-wide colors, glyphs, borders, and density resolve from the
[Theme Recipe](/theming). Per-instance labels, marks, variants, and layout
remain ordinary source in the installed Recipe.

To build your own control or design system from packaged behavior, use the
[Primitive documentation](/primitives). To understand the ownership boundary,
read [Architecture](/architecture). To compare upstream source after
installation, read [Registry](/registry#review-updates).
