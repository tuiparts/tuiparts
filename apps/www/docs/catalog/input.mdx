Input provides single-line text entry. The packaged Primitive preserves
OpenTUI's mutable buffer and event order instead of introducing browser-style
controlled rollback. The Recipe adds editable text, placeholder, cursor, and
focus colors.

## Install

<CodeGroup>

```bash React
pnpm dlx shadcn@4.13.0 add @tuiparts/react/input
```

```bash Solid
pnpm dlx shadcn@4.13.0 add @tuiparts/solid/input
```

```bash Core
pnpm dlx shadcn@4.13.0 add @tuiparts/core/input
```

</CodeGroup>

## Use the Recipe

### React and Solid

The framework Recipes export `Input` and retain the Primitive's props:

```tsx
import { Input } from "./components/ui/input";

<Input
  placeholder="Search Recipes…"
  onInput={(value) => setQuery(value)}
  onSubmit={(value) => search(value)}
/>;
```

All visual defaults are ordinary OpenTUI properties. Instance props override
the Recipe defaults because they are spread last.

### Core

The Core Recipe exports `createInput()`:

```ts
import { createInput } from "./components/ui/input";

const input = createInput(ctx, {
  placeholder: "Search Recipes…",
  onInput: (value) => setQuery(value),
  onSubmit: (value) => search(value),
});

parent.add(input);
```

The returned `InputRenderable` retains OpenTUI's native value and editing
surface.

## Customize the Recipe

The Recipe adds visual defaults only. Edit the copied file to change its
placeholder, text, cursor, and focus colors, or pass native Input properties
per instance to override those defaults. Shared values belong in the
[Theme Recipe](/theming).

## Behavior

Input preserves OpenTUI's mutable editing model. `onInput` reports buffer
changes, `onChange` reports commit-style changes, and `onSubmit` reports Enter
submission. Other editing keys and cursor behavior follow the installed
OpenTUI version. A disabled Input rejects focus, editing, and submission.

## Recipe API

Input adds no Recipe-specific fields. React and Solid forward their Primitive
props; Core forwards `InputOptions`. Native Input properties override the
Recipe's visual defaults because instance options are applied last.

Edit the Recipe to change or remove its visual defaults. When you do not want
the copied presentation layer at all, use the packaged
[Input Primitive](/primitives/input).
