Input provides single-line text entry. The packaged Primitive preserves
OpenTUI's mutable buffer and event order instead of introducing browser-style
controlled rollback. The Recipe adds editable text, placeholder, cursor, and
focus colors.

## Install

<CodeGroup>

```bash React
pnpm dlx shadcn@4.13.0 add @tuiparts/react/input
```

```bash Solid
pnpm dlx shadcn@4.13.0 add @tuiparts/solid/input
```

```bash Core
pnpm dlx shadcn@4.13.0 add @tuiparts/core/input
```

</CodeGroup>

## Use the Recipe

### React and Solid

The framework Recipes export `Input` and retain the Primitive's props:

```tsx
import { Input } from "./components/ui/input";

<Input
  placeholder="Search Recipes…"
  onInput={(value) => setQuery(value)}
  onSubmit={(value) => search(value)}
/>;
```

All visual defaults are ordinary OpenTUI properties. Instance props override
the Recipe defaults because they are spread last.

### Core

The Core Recipe exports `createInput()`:

```ts
import { createInput } from "./components/ui/input";

const input = createInput(ctx, {
  placeholder: "Search Recipes…",
  onInput: (value) => setQuery(value),
  onSubmit: (value) => search(value),
});

parent.add(input);
```

The returned `InputRenderable` retains OpenTUI's native value and editing
surface.

## Customize the Recipe

The Recipe adds visual defaults only. Edit the copied file to change its
placeholder, text, cursor, and focus colors, or pass native Input properties
per instance to override those defaults. Shared values belong in the
[Theme Recipe](/theming).

## Behavior

- `onInput(value)` reports buffer changes, including programmatic `value`
  assignments that change the buffer.
- `onChange(value)` reports commit-style changes on blur or submit.
- `onSubmit(value)` reports an Enter submission.

Editing, selection, cursor movement, insertion, and deletion follow the
installed OpenTUI version. Use native Input props for those capabilities.

`Enter` or `Return` submits the current value and fires `onSubmit`. Other
editing keys use OpenTUI's native Input behavior. A disabled Input rejects
focus, key handling, and submission.

For the exact ownership and native editing boundary, see the
[Input Primitive](/primitives/input).

## Recipe API

The React and Solid `InputProps` types extend their runtime Primitive props.
Core `InputOptions` extends the packaged `InputOptions`. The Recipe does not
add semantic convenience props; it only supplies visual defaults that callers
can override.

For the exact native option surface, use the types from your installed
`@opentui/core` package.

## Build with the Primitive

Use the packaged [Input Primitive](/primitives/input) when you want OpenTUI's
editing behavior without the Recipe's visual defaults.
