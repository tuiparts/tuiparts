Radio Group represents one selection from a set. The packaged Primitives own
the collection value, item registration, selection, roving focus, and
availability. The Recipe adds a vertical layout, one-cell marks, labels, and
colors.

Every Radio must belong to a Radio Group. Use [Checkbox](/catalog/checkbox) or
[Switch](/catalog/switch) for an independent boolean choice.

## Use the Recipe

### React and Solid

```tsx
import {
  RadioGroup,
  RadioGroupItem,
} from "./components/ui/radio-group";

<RadioGroup
  defaultValue="dark"
  onValueChange={(value, details) => {
    console.log({ value, reason: details.reason });
  }}
>
  <RadioGroupItem value="dark" label="Dark" />
  <RadioGroupItem value="light" label="Light" />
  <RadioGroupItem value="system" label="System" disabled />
</RadioGroup>;
```

Use `value` with `onValueChange` for controlled ownership, or `defaultValue`
for uncontrolled initial selection.

### Core

```ts
import {
  createRadioGroup,
  createRadioGroupItem,
} from "./components/ui/radio-group";

const colorMode = createRadioGroup(ctx, { defaultValue: "dark" });

colorMode.add(
  createRadioGroupItem(ctx, colorMode.store, {
    value: "dark",
    label: "Dark",
  }),
);
colorMode.add(
  createRadioGroupItem(ctx, colorMode.store, {
    value: "light",
    label: "Light",
  }),
);

parent.add(colorMode);
```

Each Core item receives the group's Store explicitly.

## Customize the Recipe

The copied source owns the vertical layout, one-cell mark, label assembly,
tone mapping, and convenience item API. Edit it for horizontal presentation,
wider marks, descriptions, or another public item shape. Shared defaults come
from the [Theme Recipe](/theming).

## Behavior

Radio Group supports controlled or uncontrolled single selection. Arrow keys,
Home, and End move focus and selection together; direct activation selects the
focused Radio. Unavailable items are skipped, and a disabled group rejects
item focus and selection.

## Recipe API

`RadioGroup` adds no fields in React and Solid; it forwards Primitive group
props into the starter vertical layout. Core adds `orientation` and `gap` as
editable layout options.

`RadioGroupItem` adds required `label`, optional one-cell `mark`, and the
framework-only `tone="accent" | "success"`. It also requires the Primitive
`value` used for group identity.

When the item structure does not fit, compose
[Radio Group and Radio Parts](/primitives/radio-group) directly.
