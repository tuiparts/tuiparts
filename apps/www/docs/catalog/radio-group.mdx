Radio Group represents one selection from a set. The packaged Primitives own
the collection value, item registration, selection, roving focus, and
availability. The Recipe adds a vertical layout, one-cell marks, labels, and
colors.

Every Radio must belong to a Radio Group. Use [Checkbox](/catalog/checkbox) or
[Switch](/catalog/switch) for an independent boolean choice.

## React and Solid

```tsx
import {
  RadioGroup,
  RadioGroupItem,
} from "./components/ui/radio-group";

<RadioGroup
  defaultValue="dark"
  onValueChange={(value, details) => {
    console.log({ value, reason: details.reason });
  }}
>
  <RadioGroupItem value="dark" label="Dark" />
  <RadioGroupItem value="light" label="Light" />
  <RadioGroupItem value="system" label="System" disabled />
</RadioGroup>;
```

Use `value` with `onValueChange` for controlled ownership, or `defaultValue`
for uncontrolled initial selection.

## Core

```ts
import {
  createRadioGroup,
  createRadioGroupItem,
} from "./components/ui/radio-group";

const colorMode = createRadioGroup(ctx, { defaultValue: "dark" });

colorMode.add(
  createRadioGroupItem(ctx, colorMode.store, {
    value: "dark",
    label: "Dark",
  }),
);
colorMode.add(
  createRadioGroupItem(ctx, colorMode.store, {
    value: "light",
    label: "Light",
  }),
);

parent.add(colorMode);
```

Each Core item receives the group's Store explicitly.

## Selection and focus

Keyboard navigation moves focus and selection together. Direct activation
selects the focused item. Unavailable items are skipped, and a disabled group
rejects item focus and selection.

Change details distinguish `"activation"` from `"navigation"` and report an
imperative, keyboard, or pointer source.

## Keyboard interaction

| Key | Behavior |
| --- | --- |
| Arrow keys | Moves focus and selection to the next available item. |
| `Home` | Moves focus and selection to the first available item. |
| `End` | Moves focus and selection to the last available item. |
| `Enter`, `Return`, or `Space` | Selects the focused item. |

Arrow movement follows the collection's navigation model. The starter Recipe
lays items out vertically, but Primitive keyboard handling accepts the
supported directional navigation keys independently of that visual layout.

### RadioGroupItem

`value` and `label` are required. `disabled` removes the item from available
navigation. `mark` overrides the theme radio glyph; the starter layout
reserves one terminal cell. React and Solid also expose `tone="accent" |
"success"`; the Core starter factory does not expose that tone convenience.
