title: Switch

Switch represents a setting that takes effect immediately. The packaged
Primitive owns checked state, activation, focus, and the always-mounted Thumb
Part. The Recipe chooses the track, thumb glyph, density, label, and colors.

Use [Checkbox](/catalog/checkbox) for choices commonly submitted together.

## React and Solid

```tsx
import { Switch } from "./components/ui/switch";

<Switch
  label="Dark mode"
  density="comfortable"
  symbols="round"
  defaultChecked
/>;
```

Use `checked` with `onCheckedChange` for controlled ownership, or
`defaultChecked` for uncontrolled initial state.

## Core

```ts
import { createSwitch } from "./components/ui/switch";

const colorMode = createSwitch(ctx, {
  label: "Dark mode",
  density: "comfortable",
  defaultChecked: true,
});

parent.add(colorMode);
```

The factory returns `SwitchRootRenderable` and constructs a Thumb attached to
the Root's Store.

## Anatomy

`Switch.Root` owns state and activation. `Switch.Thumb` reflects the same
readonly state and remains mounted whether checked or unchecked. The starter
Recipe absolutely positions the Thumb over a repeated track glyph; that
layout is editable and not part of the Primitive contract.

## Keyboard interaction

| Key | Behavior |
| --- | --- |
| `Enter` or `Return` | Toggles checked state. |
| `Space` | Toggles checked state. |

An uncancelled primary-pointer release and `press()` request the same change.
Disabled Switches reject focus and activation.
