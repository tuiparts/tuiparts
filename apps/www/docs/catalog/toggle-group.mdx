A group of two-state buttons that share pressed values and a single tab stop.
The recipe installs two components: `ToggleGroup`, which owns the array of
pressed values, and `ToggleGroupItem`, which renders a labeled Toggle.
Selection, roving focus, and orientation-aware navigation come from the
packaged primitives.

## Usage guidelines

- **Use a toggle group when related on/off choices should share one keyboard
  stop.** For a single independent action, use `Toggle` alone; for exactly one
  required selection, use `RadioGroup`.
- **Every grouped Toggle requires a unique string `value`.** The group
  identifies items and reports selection by those values.

## Anatomy

Import the installed components and nest items inside the group:

```tsx
import {
  ToggleGroup,
  ToggleGroupItem,
} from "./components/ui/toggle-group";

<ToggleGroup defaultValue={["left"]}>
  <ToggleGroupItem value="left" label="Left" />
  <ToggleGroupItem value="center" label="Center" />
  <ToggleGroupItem value="right" label="Right" />
</ToggleGroup>;
```

By default, one value may be pressed at a time. Set `multiple` to permit
several pressed values.

## Keyboard interactions

| Key | Behavior |
| --- | --- |
| `ŌåÉ` `ŌåÆ` | Moves focus in a horizontal group. |
| `Ōåæ` `Ōåō` | Moves focus in a vertical group. |
| `Home` | Moves focus to the first available Toggle. |
| `End` | Moves focus to the last available Toggle. |
| `Enter` | Changes the focused Toggle's pressed state. |
| `Space` | Changes the focused Toggle's pressed state. |

Focus navigation does not change selection. By default, arrow navigation
wraps at the collection edges; set `loopFocus` to `false` to stop at the
ends.
