Toggle is a two-state action that can be pressed or unpressed. The packaged
Primitive owns pressed state, activation, focus, and optional Toggle Group
membership. The Recipe adds a label, spacing, and state-dependent colors.

Use [Button](/catalog/button) for an action without retained state and
[Switch](/catalog/switch) for a labeled setting row.

## Use the Recipe

### React and Solid

```tsx
import { Toggle } from "./components/ui/toggle";

<Toggle
  label="Bold"
  defaultPressed
  onPressedChange={(pressed, details) => {
    console.log({ pressed, source: details.source });
  }}
/>;
```

Use `pressed` with `onPressedChange` for controlled ownership, or
`defaultPressed` for uncontrolled initial state.

### Core

```ts
import { createToggle } from "./components/ui/toggle";

const bold = createToggle(ctx, {
  label: "Bold",
  defaultPressed: true,
});

parent.add(bold);
```

The factory returns `ToggleRenderable`.

## Customize the Recipe

The copied source owns the label child tree, spacing, and pressed-state color
mapping. Edit it to add icons, variants, or another layout. Shared colors and
density come from the [Theme Recipe](/theming).

## Behavior

A standalone Toggle supports controlled or uncontrolled pressed state. Inside
a Toggle Group it requires a unique `value` and adopts the group's selection.
Enter, Return, Space, primary-pointer release, and imperative `press()` request
the opposite value. Disabled Toggles reject focus and activation.

Use the [Toggle Group Recipe](/catalog/toggle-group) when related Toggles
should share selection and a single keyboard stop.
