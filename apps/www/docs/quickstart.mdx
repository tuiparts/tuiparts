This tutorial installs the React Checkbox Recipe. The same Catalog address is
available under `solid/checkbox` and `core/checkbox` for the other runtimes.

You need an existing OpenTUI application and a `components.json` file. If you
have not configured a shadcn-compatible Registry before, add the tuiparts.sh
namespace first.

## Configure the Registry

Add `@tuiparts` to `components.json`:

```json title="components.json"
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "registries": {
    "@tuiparts": "https://tuiparts.sh/r/{name}.json"
  }
}
```

This maps an address such as `@tuiparts/react/checkbox` to the static Registry
item at `/r/react/checkbox.json`.

## Install Checkbox

Run the shadcn CLI from your application:

```bash
pnpm dlx shadcn@4.13.0 add @tuiparts/react/checkbox
```

The command copies `checkbox.tsx` into `components/ui`, installs its package
dependencies, and adds the consumer-owned theme files used by the Recipe.

Use `@tuiparts/solid/checkbox` for Solid or `@tuiparts/core/checkbox` for
imperative Core.

## Render the Recipe

Import the installed file through your application's local path:

```tsx
import { Checkbox } from "./components/ui/checkbox";

export function NotificationSetting() {
  return (
    <Checkbox
      label="Enable notifications"
      defaultChecked
      onCheckedChange={(checked) => console.log({ checked })}
    />
  );
}
```

Checkbox behavior comes from the packaged Primitive. Its label, mark, spacing,
and token mapping live in the installed file and are yours to edit.

## Customize it

Open `components/ui/checkbox.tsx`. You can change the layout or add Recipe
props without wrapping an opaque package API. For application-wide colors,
glyphs, borders, and density, edit `components/ui/theme.ts` instead.
