Every Recipe reads colors, glyphs, borders, and density from a consumer-owned
theme file. Installing a Recipe adds `components/ui/theme.ts`; React and Solid
also receive a small `components/ui/use-theme.tsx` binding.

There is no packaged theme runtime. The installed source contains the token
contract, subscription store, default terminal theme, and color helper. Edit
it as part of your application.

## Token contract

`Tokens` is a small semantic vocabulary shared by the starter Recipes:

```ts title="components/ui/theme.ts (excerpt)"
export interface Tokens {
  colors: {
    background: ColorInput;
    surface: ColorInput;
    foreground: ColorInput;
    mutedForeground: ColorInput;
    border: ColorInput;
    focus: ColorInput;
    primary: ColorInput;
    primaryForeground: ColorInput;
    destructive: ColorInput;
    destructiveForeground: ColorInput;
    success: ColorInput;
    successForeground: ColorInput;
    warning: ColorInput;
    warningForeground: ColorInput;
    disabled: ColorInput;
    disabledForeground: ColorInput;
  };
  glyphs: {
    check: string;
    radio: string;
    thumb: string;
    track: string;
  };
  borders: {
    style: "single" | "rounded" | "double" | "heavy";
  };
  density: {
    paddingX: number;
    comfortablePaddingX: number;
  };
}
```

Intent colors use background and foreground pairs so Recipes do not infer
text contrast. Short-lived interaction colors, such as a pressed shade or
backdrop scrim, are derived in Recipe code with `tint()`.

## Read and change the active theme

`theme.ts` exports an application-wide `theme` store:

| Member | Behavior |
| --- | --- |
| `theme.get()` | Returns the current deep-frozen token snapshot. |
| `theme.subscribe(listener)` | Subscribes to resolved theme changes and returns an unsubscribe function. |
| `theme.setActive(name)` | Activates a registered theme; returns `false` when the name is unknown. |
| `theme.setMode(mode)` | Selects `"system"`, `"dark"`, or `"light"`. |
| `theme.register(name, definition)` | Adds or replaces a theme definition. |
| `theme.override(partial)` | Applies a top-priority partial overlay, or clears it with `undefined`. |
| `theme.follow(renderer)` | Follows the renderer's terminal color mode. |
| `tint(base, overlay, alpha)` | Blends two OpenTUI colors. |

Resolved tokens merge in this order:

```text
base → active shared tokens → active mode tokens → runtime override
```

Missing keys fall through to the preceding layer.

## Use tokens in each runtime

<CodeGroup>

```tsx React
import { useTheme } from "./components/ui/use-theme";

function StatusLine() {
  const tokens = useTheme();
  return <text content="ready" fg={tokens.colors.mutedForeground} />;
}
```

```tsx Solid
import { useTheme } from "./components/ui/use-theme";

function StatusLine() {
  const tokens = useTheme();
  return <text content="ready" fg={tokens().colors.mutedForeground} />;
}
```

```ts Core
import { theme } from "./components/ui/theme";

const applyTheme = () => {
  label.fg = theme.get().colors.mutedForeground;
};

applyTheme();
const unsubscribe = theme.subscribe(applyTheme);
```

</CodeGroup>

Core code must call the styling function once before subscribing. Destroying
consumer-owned Renderables should also call the returned unsubscribe function.

## Follow terminal color mode

`"system"` refers to the terminal's detected light or dark colors, not
directly to the operating system setting. Connect the store to the renderer
once during startup:

```ts
const renderer = await createCliRenderer({ exitOnCtrlC: true });
const stopFollowingTheme = theme.follow(renderer);
```

OpenTUI derives the mode from the terminal background and emits updates when
the terminal reports a change. If no mode has been detected, the theme store
uses dark mode. Call `stopFollowingTheme()` during teardown when the renderer
does not own the full application lifetime.

## Install a preset

The Registry includes framework-neutral presets:

| Preset | Registry item | Dark variant | Light variant |
| --- | --- | --- | --- |
| Cobalt Deep | `theme-cobalt-deep` | Brand graphite and cobalt | Light brand palette |
| Catppuccin | `theme-catppuccin` | Mocha | Latte |
| Gruvbox | `theme-gruvbox` | Gruvbox Dark | Gruvbox Light |
| Rosé Pine | `theme-rosepine` | Main | Dawn |
| ASCII | `theme-ascii` | ASCII glyph overrides | Uses the base colors |

Install, register, and activate a preset:

```bash
pnpm dlx shadcn@4.13.0 add @tuiparts/theme-catppuccin
```

```ts
import { theme } from "./components/ui/theme";
import { catppuccin } from "./themes/catppuccin";

theme.register("catppuccin", catppuccin);
theme.setActive("catppuccin");
```

Presets are `DeepPartial<Tokens>` definitions. They merge over your base, so a
glyph-only preset does not replace colors and locally added tokens do not make
existing presets incomplete.

## Create a custom theme

A theme definition can provide shared tokens and optional mode overlays:

```ts title="themes/midnight.ts"
import type { ThemeDefinition } from "../components/ui/theme";

export const midnight: ThemeDefinition = {
  tokens: {
    colors: {
      background: "#0B0E15",
      primary: "#7AA2F7",
      primaryForeground: "#0B0E15",
    },
  },
  light: {
    colors: {
      background: "#E9EDF4",
      primary: "#2D62E4",
      primaryForeground: "#FFFFFF",
    },
  },
};
```

You can also edit the `terminal` base, extend `Tokens` with application
semantics, or replace the vocabulary entirely. Recipe token access is ordinary
typed property access, so TypeScript identifies the installed files that need
updating after a rename.

Persistence, theme-picker UI, and config-directory discovery are application
concerns built on `subscribe()` and `register()`; the Registry does not impose
those policies.

## Review theme updates

The Theme item follows the same consumer-owned update lifecycle as every
Recipe:

```bash
pnpm dlx shadcn@4.13.0 add @tuiparts/react/theme --diff
```

Review and merge upstream changes rather than overwriting local tokens.
